Cause









It is a common issue that some errors may occur during a team project.


1. Here is one example. In this situation, visitor can recognize that the collaborator created file "Conflicts" in the GIT_tutorial repository belonged to another user.

Solution







The easiest way is to update the local computer file. Input "git pull" before operating.


Another solution is provided by several editors such as VS code. It will automatically identify the error and offer some suggestion that which file do you want to remain.
