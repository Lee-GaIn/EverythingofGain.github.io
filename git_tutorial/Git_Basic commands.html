① Terminal and User name/email









Now user can use the terminal to input their commands. The terminal will be opened in an editor such as Visual Studio Code or GIT Bash. Download editor or right-click on the desktop and check Git bash.











User name/email









1. Before starting to use Git hub, change the user name and email are required.







2. Change user name. The command is git config --global user.name "(GitHubName)"







* If it successfully works, Git bash will not display any change







* Error will occur if the command is inappropriate.







5. Change user email. The command is git config --global user.name "(GitHubName)"







6. Now user can check both user.name and user.email.
- check user.name : git config user.name
- check user.email : git config user.email







② Clone









1. To manage software both Git hub and local computer, the user should make a clone. Open repository webpage and click code button.







2. Copy HTTPS URL







3. The command for creating a clone is git clone (HTTPS URL)







4. Git hub will create a project file on the user's desktop







5.Check file







③ Add









1. In this part, the user will know how to add new files in the local computer.







2. Currently, only README file exists in this repository.







3.Make a new file. My temporary file name is upload_file.html







4. Open the terminal and change your location. The command is cd (location)







5. Check the list of files if necessary. The command is ls







6.Then user can see your location is altered. "Desktop/". There are two commands for adding files.
- add all files in the folders: git add .
- add specific file : git add











7. Any change does not appear if the command is implemented successfully.







8. Commit user's command. Put meaningful comment inside two quotations to prepare tracking modification The command is commit -m "comment"







9. It indicates that 1 file changed which is upload_file.html







10. Upload your file to an online Git hub. The command is git push







11. Now user can see the upload file name, comment and upload time.







④Update









1. In this part, the user will understand update files. As visitor can see, GitHub lets to user add a file on the Github webpage as well as the local computer. Click "Add file" and create or upload files







2. Enter the file's name and content







3. Click "Commit new file"







4. Now user can see the new file is uploaded.







5. However, the new file does not exist in the folder on the local computer.







6. Update your local computer folder. The command is git pull







7. It shows that 1 file changed from github.com







8. Check file if necessary.
